# Yazi 使用指南：一个更现代的终端文件管理器

如果你经常在终端里处理文件，迟早会遇到一个问题：

> `cd`、`ls`、`mv`、`cp` 这一套当然能干活，但目录一深、文件一多、批量操作一复杂，人就容易烦。

这时候终端文件管理器就很有价值了。

而这几年比较顺手、也比较新的一个工具，就是 [Yazi](https://github.com/sxyazi/yazi)。

一句话先说结论：

- 如果你想要一个**快、现代、键盘优先**的终端文件管理器，Yazi 很值得试
- 如果你以前用过 `ranger`，你会很快上手
- 如果你在意预览、异步体验、扩展性和界面观感，Yazi 往往比老牌方案更讨喜

这篇就直接讲它怎么用。

## Yazi 是什么

Yazi 是一个运行在终端里的文件管理器，目标很明确：

- 快
- 交互现代
- 支持预览
- 支持批量操作
- 适合键盘流用户

它不是图形界面资源管理器，也不是 `fzf` 那种纯筛选器，而是一个更完整的 TUI 文件管理器。

你可以把它理解成：

- 有点像更现代的 `ranger`
- 也有点像命令行版 Finder / 文件资源管理器
- 但整体更偏向“为终端工作流设计”

## 为什么 Yazi 现在很火

Yazi 受欢迎，核心原因其实就几个：

### 1. 足够快

它在大目录、频繁切换目录、预览文件时的体感通常都不错。

### 2. 界面更现代

相比一些老牌 TUI 工具，Yazi 的默认观感更清爽，不那么“古早”。

### 3. 预览能力实用

图片、文本、代码、压缩包、PDF 等内容，都可以结合系统能力或外部工具做预览。

### 4. 对键盘流很友好

如果你本来就习惯 Vim / tmux / shell，Yazi 的上手成本并不高。

## 安装 Yazi

最常见的方式还是看项目 README 或 release 页面。

仓库地址：

- `https://github.com/sxyazi/yazi`

常见安装方式大概会有这些：

```bash
# macOS
brew install yazi ffmpeg sevenzip jq poppler fd ripgrep fzf zoxide imagemagick

# Arch Linux
sudo pacman -S yazi ffmpeg p7zip jq poppler fd ripgrep fzf zoxide imagemagick
```

你会发现它常常会顺带装一堆依赖，比如：

- `fd`
- `ripgrep`
- `fzf`
- `zoxide`
- `jq`
- `ffmpeg`
- `p7zip`
- `poppler`
- `imagemagick`

原因不复杂：

> Yazi 本体负责文件管理，很多“预览”“解压”“搜索”“跳转”能力依赖外部工具补齐。

所以安装完 Yazi 之后，如果你发现某些预览效果不完整，通常不是 Yazi 坏了，而是依赖没装齐。

## 启动 Yazi

最简单就是：

```bash
yazi
```

进入当前目录。

你也可以指定路径：

```bash
yazi ~/Downloads
yazi ~/Projects
```

如果你只是想先感受一下，它基本上打开就能用。

## 基础导航

Yazi 的核心操作风格很像 Vim / ranger 这一派。

常见的基本导航通常是：

- `j` / `k`：上下移动
- `h`：回到上级目录
- `l`：进入目录 / 打开项目
- `g`：跳到顶部
- `G`：跳到底部

如果你习惯方向键，通常也能工作，但真想把它用顺，还是建议尽快转成键盘流。

## 打开文件与进入目录

目录直接回车或 `l` 进入。

文件的处理取决于类型和系统配置，常见情况包括：

- 文本文件：预览或交给编辑器
- 图片：在支持的终端里做预览
- 媒体文件：调用外部程序
- 压缩包：预览结构或进一步处理

你最好把 Yazi 理解成：

> 它不是替代编辑器，而是“帮你更快地定位、筛选、批量处理文件”。

## 选择与批量操作

终端文件管理器最值钱的地方，不是“看文件”，而是“批量处理”。

常见动作通常包括：

- 选中多个文件
- 复制 / 剪切
- 粘贴
- 删除
- 重命名
- 批量移动

具体按键可能会随版本和配置有调整，所以最稳的方式还是在本机里看帮助或默认 keymap。

但你只要记住一点：

> Yazi 的真正价值，往往是替你减少一堆重复敲命令的文件操作。

比如你平时手动做这些事会比较烦：

- 从一个深目录里挑十几个文件复制到另一个目录
- 在一堆截图和素材中快速筛选有效文件
- 清理下载目录
- 快速浏览项目里的文档、图片和日志

这些都很适合用 Yazi。

## 预览能力是 Yazi 的重点

Yazi 的一大卖点就是预览体验。

常见能玩的方向包括：

- 文本预览
- 代码高亮
- 图片预览
- PDF 预览
- 视频缩略图 / 媒体信息
- 压缩包内容预览

但这里要泼一盆冷水：

> 预览体验强不强，不只看 Yazi，还看你的终端和依赖链。

比如：

- 终端是否支持图片协议
- 系统有没有安装 `ffmpeg`
- 是否装了 PDF / 图片处理工具
- shell 环境变量和 opener 配置是否正确

所以如果你看到别人演示很丝滑，而自己这边只能看纯文本，不一定是 Yazi 不行，更多时候是配套没跟上。

## 和 shell 工作流结合

Yazi 很适合和下面这些工具一起用：

- `zoxide`：目录跳转
- `fzf`：模糊查找
- `ripgrep`：全文搜索
- `bat`：更好看的文本预览
- `fd`：更快的文件查找
- `tmux`：分屏工作流
- `nvim` / `vim`：编辑器联动

典型工作流长这样：

1. 用 shell / zoxide 快速跳到项目目录
2. 用 `yazi` 浏览、筛选、定位文件
3. 找到目标后交给 `nvim` 编辑
4. 再回到 shell 做 git / build / run

所以 Yazi 最适合的人，不是“完全不会命令行的人”，而是：

> 已经在终端里干活，但想让文件操作更舒服的人。

## 配置思路

Yazi 支持配置、主题、插件和 opener 定制。

常见可调方向：

- 主题颜色
- 图标
- 默认打开方式
- 预览行为
- 插件扩展
- 快捷键映射

如果你准备长期用，建议优先折腾这三类：

### 1. opener

定义不同文件类型默认怎么打开。

比如：

- markdown 用 `nvim`
- 图片交给系统预览器
- PDF 用你常用阅读器
- 视频走外部播放器

### 2. 预览链路

把图片、PDF、压缩包、媒体文件的依赖补齐。

### 3. 主题和 UI

这个不是必须，但一旦界面顺眼，你会更愿意长期开着它。

## Yazi 和 ranger、nnn 的区别

这部分很多人都会问，我直接说直白一点。

## Yazi vs ranger

### ranger 的特点

- 老牌经典
- Vim 风格浓
- 生态和资料很多
- 很多人入门终端文件管理器就是从它开始

### Yazi 的特点

- 更现代
- 通常更快
- 预览体验和整体 UI 观感更讨喜
- 更像“新时代的 ranger 替代品”

如果你问我怎么选：

- 你已经深度绑定 ranger，而且配了很多年，没必要为了新而新
- 你现在是第一次认真用终端文件管理器，优先试 Yazi

## Yazi vs nnn

### nnn 的特点

- 非常轻
- 很快
- 极简
- 很适合喜欢“工具越小越好”的人

### Yazi 的特点

- 功能更完整
- 视觉反馈更丰富
- 预览和交互更现代
- 更适合“我要一个主力 TUI 文件管理器”的场景

简单说：

- `nnn` 更像极简派
- `Yazi` 更像现代综合派

## Yazi 适合什么人

我觉得很适合下面几类人：

- 经常在 SSH / 远程服务器上处理文件
- 喜欢 Vim / tmux / terminal workflow
- 想减少 Finder / 资源管理器和终端之间来回切换
- 想找一个能长期用的现代 TUI 文件管理器
- 喜欢折腾预览、主题和插件

## 不适合什么人

也别神化它。Yazi 也不是谁都适合。

下面这些人，可能就未必需要：

- 你几乎不用终端
- 你更依赖 GUI 拖拽
- 你只做极少量文件操作
- 你不想装额外依赖
- 你不想记快捷键

如果你本来就只是在图形界面里点点点，那装 Yazi 大概率只是新鲜两天。

## 我的建议

如果你刚开始用 Yazi，别一上来就抄别人整套高配配置。

建议按这个顺序来：

1. 先安装本体和常见依赖
2. 先熟悉基本导航、进入目录、选择文件
3. 再补图片 / PDF / 压缩包预览
4. 再去折腾主题、插件、opener
5. 最后再考虑和 `zoxide`、`nvim`、`tmux` 深度联动

这样比较不容易把自己绕晕。

## 总结

Yazi 之所以值得写，不是因为它“新”，而是因为它确实把终端文件管理这件事做得更现代、更顺手了。

如果你以前觉得：

- `ranger` 有点老
- `nnn` 太素
- 纯 shell 管文件太累

那 Yazi 基本就是一个很自然的答案。

一句话收尾：

> Yazi 不一定是所有人的终端文件管理器，但它很可能是当下最好上手、也最容易长期留下来的那一个。
